Lesson 1: Introduction

HTML (HyperText Markup Language) is the standard markup language used to create and structure content on the web, serving as the fundamental building block of the World Wide Web. Developed by Tim Berners-Lee in 1990 (or 1991 according to some sources), HTML defines the meaning and organization of web content rather than its visual appearance, which is handled by CSS and JavaScript. HTML is not a programming language but a markup language that uses tags enclosed in angle brackets (e.g., ) to annotate text, images, and other elements. These tags form a tree of nodes that instruct web browsers on how to display information, such as headings, paragraphs, links, lists, and media.

Lesson 2: HTML Basics

HTML (HyperText Markup Language) is the standard markup language used to structure and define the content of web pages, utilizing elements and tags to organize text, images, links, and multimedia. It serves as the foundation of nearly all websites, with over 95% of web pages relying on it today. Basic Document Structure Every HTML document requires a specific skeleton to function correctly in a browser:

  • !DOCTYPE html: Declares the document type and version (HTML5) at the very top.
  • html: The root element that encloses the entire document.
  • head: Contains metadata (like the title, character encoding, and linked stylesheets) that is not displayed on the page.
  • title: Defines the text shown in the browser tab or window.
  • body: Encloses all visible content (text, images, headings) that users see on the webpage.

Common Basic Tags Developers use specific tags to define different parts of a page's layout:

  • Headings: h1 to h6 for titles, where h1 is the most important.
  • Paragraphs: p for blocks of text.
  • Links: a for creating hyperlinks.
  • Lists: ul for unordered lists and ol for ordered lists.
  • Images: img for embedding images.
  • Formatting: Various tags for styling text (e.g., strong, em, code).

Lesson 3: CSS Styling

CSS (Cascading Style Sheets) is a style sheet language used to describe the presentation of HTML documents, allowing developers to control colors, fonts, sizes, spacing, layout, and background images. It is a rule-based language where you define selectors to target specific HTML elements and apply declarations containing property-value pairs within curly braces. There are three primary methods to add CSS to an HTML document:

  • Inline CSS: Applied directly to an HTML element using the style attribute (e.g., p style="color: red;").
  • Internal CSS: Placed within a style tag in the head section of the document, allowing you to style multiple elements without affecting external files.
  • External CSS: Linked via a separate .css file using the link>tag in the head, enabling reuse across multiple pages and better organization.

CSS rules follow a specific syntax where the selector identifies the element to style, followed by a block containing properties and their values separated by colons and semicolons. For example, p { color: red; } selects all paragraph elements and sets their text color to red. The cascading nature of CSS means that styles applied to parent elements (like body) automatically apply to child elements unless overridden by more specific rules.

Lesson 4: JavaScript

JavaScript (JS) is a high-level, multi-paradigm programming language and a core technology of the World Wide Web, alongside HTML and CSS. Created by Brendan Eich in 1995, it is used by 98.9% to 99% of all websites to enable dynamic client-side behavior, such as interactive elements, animations, and real-time content updates.

Key characteristics and capabilities include:

  • Execution: It is an interpreted (or just-in-time compiled) language that runs on JavaScript engines within web browsers and on servers via runtimes like Node.js.
  • Paradigms: It supports event-driven, functional, imperative, and object-oriented programming styles, utilizing prototypal inheritance (though ES6 introduced class syntax as syntactic sugar).
  • Typing: It features dynamic, weak, duck typing and garbage collection, meaning variable types are determined at runtime and can change.
  • Asynchronous Programming: It supports asynchronous operations, allowing non-blocking execution of code, particularly useful for handling I/O operations and API calls.
  • Libraries and Frameworks: A rich ecosystem of libraries and frameworks (e.g., React, Angular, Vue) exists to facilitate rapid development of complex web applications.
  • Performance: Modern JavaScript engines offer high performance through optimization techniques like Just-In-Time (JIT) compilation.

While TypeScript exists as a superset that adds static typing to JavaScript, the core language remains the dominant client-side scripting language, capable of handling everything from simple page interactions to complex Single Page Applications (SPA) and full-stack development.

Core Functionality & How It Works:

  • Client-Side Execution: JavaScript code is executed directly in the user's web browser. This allows for immediate interaction (like form validation or animations) without constant communication with the server, leading to faster, more responsive applications.
  • he DOM: It interacts with the Document Object Model (DOM), which is a programming interface for web documents. JavaScript can dynamically change a page's structure, style, and content by accessing and modifying the DOM.
  • Event-Driven: JavaScript is fundamentally event-driven. It can respond to user interactions like clicks, keystrokes, or page loads, triggering specific functions to execute.
  • Dynamic Typing: Variables in JavaScript are not bound to a specific data type, allowing for flexibility during development (e.g., a variable can hold a number and later be reassigned to a string).
